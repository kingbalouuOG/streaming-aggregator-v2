# Sync Pipeline Runbook

Bulk content sync from TMDb → SA API → OMDB → Vectors. Run on demand for backfill or initial population. The daily incremental sync runs automatically via Edge Function.

## Stages

```
npx tsx scripts/sync-content.ts --stage tmdb
npx tsx scripts/sync-content.ts --stage sa
npx tsx scripts/sync-content.ts --stage omdb
npx tsx scripts/sync-content.ts --stage vectors
```

| Stage | Source | Writes | Idempotent | Notes |
|---|---|---|---|---|
| `tmdb` | TMDb `/discover` | `titles` (insert + update), also computes legacy `content_vector` for any title where it is NULL | yes | Iterates by `popularity.desc` until target row count. Default ~20K titles. |
| `sa` | Streaming Availability API | `streaming_availability` (per-service rows) | yes | Iterates by tmdb_id list from `titles`. Honours `Retry-After`. Concurrency = 4. |
| `omdb` | OMDB | `titles.imdb_rating`, `titles.rt_score` | yes | One call per IMDb ID. Skips titles missing IMDb ID. Free-tier quota: 1000/day. |
| `vectors` | derived from existing `titles` | `titles.content_vector` (legacy 24D) | yes | Backfill-only. Newer `embedding vector(1536)` is generated by `embed-new-titles` Edge Function and `scripts/embeddings/backfill-embeddings.ts`. |

## Prerequisites

- `.env` populated: `VITE_TMDB_API_KEY`, `VITE_OMDB_API_KEY`, `SA_API_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, `VITE_SUPABASE_URL`.
- Supabase project healthy.
- Disk space for response cache under `node_modules/.cache/` (small).

## Common failures

| Symptom | Cause | Fix |
|---|---|---|
| 429 from SA API mid-run | RapidAPI quota throttled | Wait until `Retry-After` window; script auto-resumes. If failing repeatedly, check tier in RapidAPI dashboard. |
| `EPERM` on Windows during write | Antivirus or VS Code file watcher | Close watchers / antivirus exclusion for the repo. Parking lot IN-XPS-005. |
| Empty `streaming_availability` writes | Missing `service_role` key | Confirm `SUPABASE_SERVICE_ROLE_KEY` set; sync requires service-role to bypass RLS for writes. |
| `vectors` stage runs but updates 0 rows | All titles already have `content_vector` | Expected. Stage only backfills NULLs. |

## Daily incremental sync

Runs automatically via pg_cron at 06:00 UTC (migration 006). Edge Function path: `supabase/functions/sync-incremental/`. Hits SA API `/changes`, writes deltas to `streaming_availability`, appends to `streaming_history`.

To run manually:

```bash
supabase functions invoke sync-incremental --no-verify-jwt
```

(`--no-verify-jwt` because the function trusts the project's service-role context internally.)
