# Source: ENG-1 Eval (2026-06-10, runs 1 + 2)

Authoritative gate-result record for Phase ENG-1. Harness: `npm run eval:eng1` (`scripts/evaluation/eng1-eval.ts`) — self-contained, in-memory centroids via the production pure modules + live `match_titles_by_vector` RPC, read-only, deterministic, with cold-HNSW warmup + retry. Gate definitions: E&P brief §3.6 + ENG-1 plan §6.

## Results

| Section | Decision / result |
|---|---|
| A — τ merge threshold | **Keep 0.80.** Pairwise cosine over the 16 cluster seed centroids: max 0.7532 (anime × family) — zero pairs merge at 0.80; the K ≤ 3 cap does the structural merging. τ = 0.75 would always merge anime × family even for deliberate 2-cluster pickers. (Plan's "24 clusters" corrected: taxonomy has 16.) |
| B — Coverage | **PASS** — synthetic feel-good + anime profile (k-means K = 3, weights [0.467, 0.333, 0.200]): merged top-20 spans **3** source interests (gate ≥ 2); single-centroid baseline leans 2. |
| B — Recall@500 | **Inconclusive at n=2** (0/2 vs 0/2 — technically passes multi ≥ single, do not quote). Bounded by thin cluster rep lists. |
| C — Avoid-set γ sweep | **Keep 0.15.** Thumbs-downing the pool's #1: full suppression of measured neighbours at every γ ∈ {0.10, 0.15, 0.20}, zero positive regression — insensitive in the range. |
| Run 2 — parity | **PASS** against the live ENG-1 Edge + regenerated golden: determinism across consecutive calls (validates the daily-seeded exploration slot), zero filter leaks, zero cross-row dupes. |
| Run 2 — real-profile recall | **Skipped by design** — prototype account has 8 distinct positives, below the 10-title floor. Carried forward as a data-accumulation item; ENG-2's data gate covers it at scale. |

**Phase-exit verdict:** structural gates (coverage, parity, suppression, τ, γ) all green; recall gates carry forward, not blocking. On-device behaviour confirmed by Joe (interest mixing, exploration slots at 6/14, avoid suppression, instant slider re-rank).

## Findings filed

1. **Cluster rep lists are thin — curation, not coverage** (reclassified after a rep-ID audit: 67/68 rep ids in catalogue WITH embeddings; catalogue coverage 98.63%; 13/16 clusters define only 2–4 reps) → IN-PX-55, Joe-owned.
2. Impressions lack `media_type` — 0.8% training-extract collision class → IN-PX-56.
3. Cold HNSW index blows the statement timeout on first RPC after idle — harness warms + retries; PLAT-3's always-warm Worker kills the class.

Run 2 also surfaced three UI bugs on device (2 slider labels + fingerprint-card contentMix axis inversion) — fixed in commit `ab79688`.
