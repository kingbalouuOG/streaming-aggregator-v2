# Source: Design Reference v0.1

Visual reference index for CC implementation work. Maps implementation areas to PNG files in `docs/v2/design-references/`.

## What it pins

- Bottom Navigation: `Menu_BottomNav.png` plus three iPhone-context variants.
- Onboarding Flow: `V2_Onboarding_Step_1.png` through `V2_Onboarding_Step_5.png`.
- Home Surface: `Home.png`.
- For You Surface: `For_You.png`.
- Profile area: 13 PNGs covering landing + sub-pages + modals.
- Sliders: appear in onboarding Step 5, Profile sub-page, For You modal/tray. Visual treatment consistent across all three.

## Phase mapping

- Phase 0: Bottom Navigation, Detail Page (no design — CC discretion).
- Phase 3: Onboarding (all 5 steps), Profile area, Sliders.
- Phase 4: Home, For You, Slider parameter wiring.
- Phase 4.5: For You mood room card pattern.

## What's not designed

"Not Interested" detail page button — CC discretion within constraints from Detail Page Signal Capture Spec §2.7. Empty/error/loading states. Animation specs. Accessibility-specific visual treatments. Future v2.5 designs.

## Why it matters

Visual baseline for every screen. Strategy docs are authoritative for behaviour and copy; the design reference is authoritative only for layout, structure, and visual hierarchy.
